Wire `VENFORK_PUSH_TOKEN` into the scheduled-sync workflow so the GitHub Actions
runner can push cross-repo from the private mirror to the public fork.

- The workflow generated by `venfork schedule set` now uses
  `actions/checkout@v4` with `token: ${{ secrets.VENFORK_PUSH_TOKEN || github.token }}`
  and `fetch-depth: 0`. The `||` fallback keeps existing setups behaving
  identically when the secret is unset.
- Adds a step that rewrites SSH GitHub URLs to HTTPS so the extraheader auth
  installed by `actions/checkout` applies to the SSH-form remote URLs that
  `venfork-config` typically stores.
- `venfork schedule set` now prints the exact `gh secret set VENFORK_PUSH_TOKEN`
  command after success, with the resolved mirror path filled in.
- README documents the new secret under `venfork schedule set`.

Without setting the secret, behaviour is unchanged from the previous version
(scheduled runs continue to fail at the public-fork push, same as before).
With the secret set, scheduled sync actually completes end-to-end.
